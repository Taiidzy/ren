# FAQ

Часто задаваемые вопросы о Ren и ответы на них.

## Общие вопросы

### Что такое Ren?

**Ren** — кроссплатформенный мессенджер со сквозным шифрованием (E2EE). Все сообщения шифруются на устройстве отправителя и расшифровываются только на устройстве получателя. Сервер не имеет доступа к содержимому переписки.

### Какие платформы поддерживаются?

- **Android** — полностью поддерживается
- **iOS** — полностью поддерживается
- **Web** — полностью поддерживается (React + WASM)
- **Desktop** — в разработке (через Flutter)

### Это бесплатно?

Да, Ren — полностью бесплатный проект с открытым исходным кодом (Apache 2.0).

### Где хранятся мои данные?

- **Сообщения** — зашифрованы на сервере, расшифровываются только на устройствах
- **Файлы** — зашифрованы на сервере, кэшируются локально
- **Ключи** — приватные ключи никогда не покидают устройство
- **Пароль** — хранится только хеш (Argon2id)

## Безопасность

### Насколько безопасно Ren?

Ren использует современные криптографические алгоритмы:

- **X25519** — обмен ключами (ECDH)
- **Ed25519** — подпись публичных ключей
- **ChaCha20-Poly1305** — шифрование сообщений (AEAD)
- **PBKDF2-HMAC-SHA256** — деривация ключа из пароля (100k итераций)
- **Argon2id** — деривация ключа восстановления (memory-hard)

### Может ли сервер прочитать мои сообщения?

**Нет.** Сообщения шифруются на устройстве отправителя и расшифровываются только на устройстве получателя. Сервер хранит только зашифрованные данные.

Исключение: **групповые и каналные сообщения** в текущей версии **НЕ защищены E2EE**. План исправления: Sender Keys protocol.

### Что будет если я забуду пароль?

Вы можете восстановить доступ через **ключ восстановления** (12-словная мнемоническая фраза). Ключ генерируется при регистрации и должен быть сохранён пользователем.

> **Важно:** Сервер не хранит ваш пароль и не может восстановить доступ без ключа восстановления.

### Можно ли использовать Ren без интернета?

Нет, для обмена сообщениями требуется подключение к серверу. Однако вы можете:
- Просматривать кэшированные сообщения
- Писать сообщения (отправятся при подключении)

### Есть ли верификация ключей?

Да, реализована криптографическая верификация публичных ключей через Ed25519 signature (P0-2). При получении публичного ключа проверяется подпись.

## Функционал

### Какие типы сообщений поддерживаются?

- **Текстовые сообщения** — с E2EE
- **Файлы** — до 50MB, с E2EE
- **Изображения** — с предпросмотром, с E2EE
- **Голосовые сообщения** — с E2EE
- **Видео** — с E2EE

### Есть ли групповые чаты?

Да, поддерживаются:
- **Группы** — до 256 участников
- **Каналы** — неограниченное количество подписчиков

> **Важно:** Групповые/каналные сообщения НЕ защищены E2EE в текущей версии.

### Можно ли редактировать сообщения?

Да, сообщения можно редактировать в течение 24 часов после отправки. Изменённые сообщения помечаются меткой времени редактирования.

### Можно ли удалять сообщения?

- **Свои сообщения** — можно удалить в любое время
- **Чужие сообщения** — только если вы admin/owner в групповом чате

### Есть ли пересылка сообщений?

Да, можно пересылать сообщения между любыми чатами с сохранением метаданных источника.

### Есть ли ответы на сообщения?

Да, можно отвечать на конкретные сообщения с ссылкой на исходное.

### Есть ли индикаторы прочтения?

Да:
- ✓ — отправлено (сохранено на сервере)
- ✓✓ — доставлено (получено собеседником)
- ✓✓ (цветные) — прочитано (собеседник открыл чат)

### Есть ли typing indicators?

Да, собеседник видит когда вы набираете текст. Можно отключить в настройках.

### Есть ли темная тема?

Да, поддерживаются светлая и тёмная темы с автоматическим переключением.

## Технические вопросы

### Как работает E2EE в Ren?

1. При регистрации генерируется пара ключей X25519
2. Публичный ключ отправляется на сервер
3. Приватный ключ шифруется мастер-ключом (из пароля) и ключом восстановления
4. При отправке сообщения:
   - Генерируется случайный симметричный ключ
   - Сообщение шифруется симметричным ключом
   - Симметричный ключ обёртывается публичным ключом получателя
5. Получатель расшифровывает конверт своим приватным ключом и получает симметричный ключ
6. Сообщение расшифровывается симметричным ключом

### Почему Rust?

Rust выбран по причинам:
- **Безопасность памяти** — нет segfault, buffer overflow
- **Производительность** — сравнима с C/C++
- **Кроссплатформенность** — компиляция во все платформы
- **Единая кодовая база** — один SDK для всех платформ

### Можно ли использовать свой сервер?

Да, проект открыт и может быть развёрнут на своём сервере. См. [Развёртывание](/docs/deployment).

### Есть ли API для разработчиков?

Да, backend предоставляет REST API и WebSocket. См. [API Reference](/docs/api/reference).

### Как интегрировать Ren в своё приложение?

Используйте Ren-SDK:

```rust
// Rust
use ren_sdk::{generate_key_pair, encrypt_message, decrypt_message};

let key_pair = generate_key_pair();
let encrypted = encrypt_message("Hello!", &key);
```

```typescript
// TypeScript/WASM
import { generateKeyPair, encryptMessage } from 'ren-sdk';

const keyPair = generateKeyPair();
const encrypted = encryptMessage("Hello!", key);
```

### Какие есть ограничения?

- Файлы до 50MB
- Групповые сообщения без E2EE (пока)
- Нет голосовых/видео звонков (в планах)
- Нет ботов (в планах)

## Устранение проблем

### Сообщение не отправляется

Проверьте подключение к интернету. Приложение автоматически переподключится при восстановлении соединения.

### Файл не загружается

Проверьте размер файла (макс. 50MB). Если файл больше, разделите его на части.

### Приложение тормозит

Очистите кэш: Настройки → Хранилище → Очистить кэш.

### Не приходит уведомление

Проверьте настройки уведомлений:
- Настройки → Уведомления → Включить
- Проверьте системные настройки приложения

### Забыл ключ восстановления

К сожалению, без ключа восстановления невозможно восстановить доступ при потере пароля. Сервер не хранит ключи и не может помочь.
